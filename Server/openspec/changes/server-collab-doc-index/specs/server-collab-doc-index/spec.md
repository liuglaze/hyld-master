## ADDED Requirements

### Requirement: 服务端对外协作文档必须提供任务型快速索引
`Server/CLAUDE.md` MUST 以外部协作场景为主线提供快速索引，至少覆盖登录、房间、匹配、战斗帧同步四类场景，并为每个场景给出服务端入口文件、协议通道与观测点。

#### Scenario: 外部协作者定位登录链路
- **WHEN** 客户端协作方需要排查登录异常
- **THEN** 文档 MUST 在索引表中直接给出登录场景对应的服务端入口、路由文件和日志观测点

#### Scenario: 外部协作者定位帧同步链路
- **WHEN** 联调人员需要排查帧不同步
- **THEN** 文档 MUST 在索引表中给出 TCP/UDP 分工、战斗入口和关键帧观测字段

### Requirement: 服务端协作文档必须前置跨端联调入口
`Server/CLAUDE.md` MUST 在前部章节给出跨端联调最小清单，包括请求标识、关键日志键、常见失败点与排查顺序。

#### Scenario: 首次联调准备
- **WHEN** 新同学首次参与客户端与服务端联调
- **THEN** 文档 MUST 提供可直接执行的联调清单，而不要求先通读完整架构章节

### Requirement: 协作文档必须明确已知运行时风险边界
`Server/CLAUDE.md` MUST 明确列出已知高风险点（包括 UDP 单例回调并发串扰风险）以及对应的协作注意事项。

#### Scenario: 跨端提案涉及战斗链路
- **WHEN** 新提案涉及战斗 UDP 路径
- **THEN** 文档 MUST 指示提案阶段需要评估并记录单例回调串扰风险
